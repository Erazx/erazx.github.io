##第1个Django应用，Part 1

通过本教程，我们将带你创建一个简单的民意调查应用，它包含两部分：

  • 一个公共页面供人们查看投票结果跟投票。
  • 一个管理页面供你添加、修改跟删除投票。

我们假定你已经安装好Django了，你可以通过以下命令查看Django是否已安装以及Django的版本：

python -c “import django; print(django.get_version())”

若Django已安装，将输出Django的版本信息。若未安装，则会报”No module named django”的错误。

本教程基于Django 1.9 + Python 3.4。若使用Python 2.7 ，则需要根据备注相应调整代码。

参照How to install Django文档，了解如何更新Django版本。

创建一个项目

若是首次使用Django，你需要注意下一些初始化设置。即你需要自动生成Django项目代码–一个Django实例，包括数据库设置，Django专用选项，应用程序专业配置项。
在命令行终端，执行cd命令，进入存放代码的目录，然后执行一下命令创建Django项目：

django-admin startproject mysite

该命令将创建一个mysite的项目目录。若无效，可以查看Problems running django-admin

Note

要避免使用python或django内建元素命名项目。尤其是应该避免使用django（跟django自己就冲突了）或者test（跟python的一个内建包冲突了）类似的名称命名项目。

代码应该放哪里

如果你的后台代码是纯php的（没有使用任何现代框架）,你应当将代码放到web目录下（如/var/www）。而使用Django，则不建议放任何Python的代码到web目录，因为通过web有可能获取到该代码，不利于web安全。
建议将代码放到web目录外，如 /home/mycode

暂时忽略数据库迁移未应用的警告，后面我们再处理数据库。

至此，你启动了一个Django开发服务器，基于纯Python写的轻量级web服务器。这种方式可以使你快速进入Django实际开发，而不需要去搭建一个生产环境服务器（如 Apache）。

要注意，在生产环境中不要用这种方式启动服务。

现在服务已经启动了，访问 http://127.0.0.1:8000 就可以看到”Welcome to Django”页面了。

修改端口

默认情况下,runserver 命令会启动服务器，侦听在127.0.0.1的8000端口.

可以传递端口参数，修改服务侦听端口，如改为 8080 端口

python manage.py runserver 8080

或者改为侦听公网ip：

python manage.py runserver 0.0.0.0:8000

runserver 完整的文档请看runserver

自动重载runserver

runserver 对每次请求都会根据需要重新加载Python代码，所以修改了代码后并不需要重启runserver使修改生效。但是，新增代码文件不会触发重启动作，故像新增文件这种操作需要重启下runserver。

创建民意调查应用

每个基于Django的应用程序都是由一个符合某些约定的Python包组成。Django 提供了自动生成程序基本目录结构的工具，从而我们可以专注于代码编写而不是创建目录。

项目 Vs 应用

项目与应用的区别，在于app是一个web程序，它能完成某件事情，提供某种服务，如一个博客系统，一个公共记录数据库，或一个简单的投票系统。而项目是特定网站的一系列配置选项跟app的集合。一个项目可以包含多个app，而一个app也可以属于多个项目。

你的app可以放在Python path下的任何地方。本教程中，我们将在manage.py文件所在目录中创建我们的app，从而可以从顶级目录中导入app模块，而不需要从mysite的子目录中导入。
